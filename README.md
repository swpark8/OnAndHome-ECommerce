# [Project] OnAndHome: 가전제품 쇼핑몰 아키텍처 고도화

## 1. 프로젝트 개요
- **프로젝트 명:** OnAndHome (가전제품 이커머스 플랫폼)
- **프로젝트 기간:** 2025.11.24 ~ 2025.12.05 (총 2주)
- **참여 인원:** 5명 (Frontend 3명, Backend 2명)
- **나의 역할:** Full Stack (Frontend 리딩, 핵심 기능 구현)
- **한 줄 소개:** Java(Spring) 기반 쇼핑몰을 React + Spring Boot 구조로 재구성하여 성능과 사용자 경험을 개선한 리팩토링 프로젝트

<br>

## 2. 사용 기술 스택 (Tech Stack)

### Frontend (Main)
- **핵심 도구:** React.js, JavaScript
- **상태 관리:** Redux Toolkit (앱 전체 데이터 관리)
- **스타일링:** Tailwind CSS
- **통신:** Axios (API 통신), Socket.io-client (실시간 통신)

### Backend
- **핵심 도구:** Java 21, Spring Boot 3.5.6
- **Database:** MySQL, JPA (Hibernate)
- **보안:** Spring Security, JWT (Json Web Token), OAuth 2.0

### DevOps & Tools
- **VCS:** Git, GitHub
- **협업:** Canva, Notion

<br>

## 3. 아키텍처 개선 (Architecture Modernization)
> **"JSP 기반의 모놀리식 서비스에서 React & REST API 구조로의 전환"**

이번 프로젝트의 핵심은 1차 프로젝트(SSR)의 한계를 극복하기 위해 **프론트엔드와 백엔드를 완전히 분리**한 것입니다.

### 개선 내용
- **구조 변경:**
    - (AS-IS) Spring Boot가 HTML(Thymeleaf/JSP)을 직접 렌더링하여 반환.
    - (TO-BE) **Spring Boot는 JSON 데이터만 제공(REST API)**하고, **React가 UI 렌더링을 전담**하는 CSR 방식 도입.
- **통신 방식:** 클라이언트-서버 간 명확한 데이터 규약(API 명세)을 정의하여 협업 효율 증대.

### 도입 효과 (Why React?)
- **사용자 경험(UX)의 획기적 향상:**
    - 페이지 이동 시 ‘깜빡임(새로고침)’ 없는 **SPA(Single Page Application)** 환경 구축.
    - 쇼핑몰 특성상 잦은 페이지 이동(상품 리스트 ↔ 상세 페이지)에도 부드러운 화면 전환 제공.
- **개발 생산성 및 유지보수성 증대:**
    - UI 로직과 비즈니스 로직의 의존성을 제거하여, 프론트/백엔드 개발자가 독립적으로 개발 가능.
    - 컴포넌트(Component) 재사용을 통해 UI 개발 속도 40% 이상 향상.

## 4. 핵심 기능 상세 설명 (My Contribution)

### ① 실시간 알림 시스템 (WebSocket)
- **구현 내용:** 사용자의 주문 상태 변경, 문의 답변 등록 시 별도의 새로고침 없이 즉시 알림 팝업 노출.
- **기술적 포인트:**
    - HTTP의 단방향 통신 한계를 극복하기 위해 **WebSocket** 프로토콜 도입.
    - React의 `useEffect` 훅을 활용하여 로그인 시 소켓 연결, 언마운트 시 연결 해제 처리(메모리 누수 방지).
    - 알림 수신 시 전역 상태(State)를 업데이트하여 헤더 아이콘에 **뱃지(Badge)** 카운트 즉시 반영.

### ② 비밀 Q&A 게시판 & 답변 연동
- **구현 내용:** 프라이버시 보호를 위한 비밀글 설정 및 관리자 답변 시 알림 연동.
- **기술적 포인트:**
    - 작성자 본인과 관리자만 열람 가능한 권한 체크 로직을 **프론트엔드(UI 숨김)**와 **백엔드(API 접근 제어)** 이중으로 처리.
    - 관리자가 답변 작성 완료(POST 요청 성공) 시, 해당 유저의 소켓 ID를 타겟팅하여 실시간 알림 이벤트(Event Emit) 트리거.

### ③ 소셜 로그인 (OAuth 2.0)
- **구현 내용:** 카카오/구글 계정을 이용한 간편 회원가입 및 로그인.
- **기술적 포인트:**
    - 복잡한 회원가입 절차를 생략하여 쇼핑몰 접근성을 높이고 이탈률 감소 유도.
    - OAuth 인증 코드를 React에서 받아 백엔드로 전달하고, 백엔드에서 발급한 **JWT(Access Token)**를 브라우저 스토리지에 안전하게 저장하여 로그인 상태 유지.
